# Subagent LLM-judge prompt template (T1 mechanism)

This is the prompt template used to spawn a haiku/sonnet subagent that
acts as an external supervision signal on borderline test rows. The
subagent reads ONLY the prompt below — no CLAUDE.md, no parent context
needed. Designed to fit comfortably under haiku's 200k input window.

## When to use

A row is "borderline" when ≥2 of these hold:
- Multiple LB-validated submissions (≥3 of {4b, idea5, B, v1_rf,
  rawashishsin}) disagree on the class.
- DGP rule score is in {3, 4, 5, 6, 7} (boundary-band).
- 14-bank max-prob < 0.85 on this row.

Identify ~100–500 such rows. Pass them to subagents in batches
(see `scripts/T1_*.py` orchestrator below).

## Subagent prompt template (DO NOT MODIFY without re-testing)

```
Task: classify N rows of synthetic irrigation-need data as
Low / Medium / High.

Background: synthetic data generated by a NN trained on a 10k-row
real-world dataset. We have reverse-engineered an integer rule that
matches the labels ~98.4% of the time:

  dry     = (Soil_Moisture < 25)         # 0 or 1
  norain  = (Rainfall_mm   < 300)        # 0 or 1
  hot     = (Temperature_C > 30)         # 0 or 1
  windy   = (Wind_Speed_kmh > 10)        # 0 or 1
  nomulch = (Mulching_Used == "No")      # 0 or 1
  Kc      = 2 if Crop_Growth_Stage in {Flowering, Vegetative} else 0
  score   = 2*(dry + norain) + (hot + windy + nomulch) + Kc
  Low if score <= 3 ; Medium if 4 <= score <= 6 ; High if score >= 7

About 1.6% of rows are "flipped" by the NN to a neighbouring class
based on non-rule features (Humidity, Soil_pH, Previous_Irrigation_mm,
Electrical_Conductivity, Organic_Carbon, Sunlight_Hours,
Field_Area_hectare, Crop_Type, Region, Season, Soil_Type,
Irrigation_Type, Water_Source). The NN over-irrigates when humidity
is low + recent irrigation small + organic carbon low.

For each row I give you, reply in this EXACT format (one block per row):

ROW <test_id>
RULE_SCORE: <int>
RULE_PRED: <Low|Medium|High>
FINAL: <Low|Medium|High>
CONF: <0.0-1.0>
REASON: <≤15 words>

ROWS TO CLASSIFY:
{batch_of_rows}
```

## Batch row format (passed in `{batch_of_rows}`)

Each row formatted as a single line of `key=value` pairs:

```
ROW 634757 Soil_Moisture=21.82 Rainfall_mm=292.5 Temperature_C=26.47 Wind_Speed_kmh=4.32 Mulching_Used=Yes Crop_Growth_Stage=Vegetative Soil_pH=6.99 Humidity=62.28 Soil_Type=Sandy Organic_Carbon=0.55 Electrical_Conductivity=1.41 Sunlight_Hours=5.12 Crop_Type=Maize Season=Kharif Irrigation_Type=Drip Water_Source=Groundwater Field_Area_hectare=8.56 Previous_Irrigation_mm=104.56 Region=Central
```

## Batch-size estimates

- Each row: ~250 tokens
- Header (rule + format): ~400 tokens
- Reply per row: ~50 tokens
- Total per batch of 50 rows: ~13k input + ~2.5k output → ~15k tokens

Haiku context allows comfortably ≥1000 rows per batch, but Anthropic's
output-token limit per call is ~8k → cap batches at 100–150 rows.

## Decision rule (T1 4th-axis override)

After collecting all rows' `FINAL` labels, build a candidate that
overrides 4b's argmax ONLY when all of:
- LLM `FINAL` ≠ 4b's argmax on this row
- LLM `CONF` ≥ 0.7
- 14-bank-majority agrees with LLM `FINAL`
- Direction is H→M (LB-validated direction; M→H historically NULLs)

Validate on TRAIN OOF before LB probe. Require:
- TRAIN OOF macro lift ≥ +0.0001 vs 4b OOF analog
- H→M precision ≥ 92% (break-even under macro-recall)
