## Headline

**The +0.00086 LB jump from the meta-stacker is *probably real signal*,
not a hidden leak.** Negative OOF→LB gap pattern fits prior submissions
on this comp (digit-XGB −0.00019, digits-OTE blend −0.00005), suggesting
test distribution is mildly easier than CV folds. The first sub-agent's
"confirmed critical leakage" framing of `iso_cal` on full OOF is

But the audit *does* surface real weaknesses. Listed below.

**Recommended swap**: `submission_3way_recipe025_s1035_s7040.csv`. Half
the premium of current hedge, sidesteps the meta-stacker + iso layers

**Alternative**: `submission_recipe_full_te_catboost.csv` if you'd

The team's documented reasoning for the current hedge ("primary already
includes XGB + RealMLP + iso-nonrule + meta-stacker, so model-family
diversity is maxed *inside primary*") is a category mistake: hedge needs
to differ from primary, not duplicate primary's internal diversity. If
recipe-XGB pipeline itself fails on private, intra-primary diversity
won't save it.

### F2 (MEDIUM) — `iso_cal` fits on full OOF, applied as feature into meta-stacker

`scripts/tier1b_xgb_metastack.py:112` and `scripts/tier1b_greedy_with_meta.py:91, 121`
fit `IsotonicRegression` on `(full_oof_prob_class_c, y == c)` then apply
to OOF + test. The OOF transform uses an iso fit on its own data — a
classic stacking-with-shared-fold overfit path.

Magnitude: 1-D monotonic mapping over 630k points, per-row leak ~1/N.
Net OOF inflation likely 0.0001-0.0003. Honest fix is per-fold isotonic
(fit on tr_idx OOF, transform va_idx OOF). Doesn't affect the submitted
CSV (test predictions correct in expectation), but the OOF lifts quoted
in the log are mildly inflated relative to a clean-CV-honest version.

**No action for the deadline**; flag for future-comp playbook.

### F3 (LOW-MEDIUM) — Standard "shared-fold stacking" leak in meta-stacker

`tier1b_xgb_metastack.py` trains a 5-fold XGB meta-stacker using the
same StratifiedKFold(seed=42) as every base component. For meta's
training fold-k row j ∈ folds {0..3}, the feature `oof_A[j]` was
generated by A's model trained on folds {!=fold_of_j}, which *includes
fold k*. Meta is trained on features generated by models that saw its
val labels.

This is a documented pitfall (sklearn / mlxtend stacking has the same
property). Practical magnitude is small because A's models are nearly
identical when one fold differs. **No action.**

### F4 (MEDIUM) — Selection bias from ~30 cumulative LB probes

With per-submission noise σ ≈ 0.0005 on macro-recall (135k public split,
3.3% High class), max-of-30 order statistic ≈ 1.5σ above true mean.

Estimated true LB of primary ≈ 0.98094 − 0.00075 ≈ **0.98019**, with
private LB centered there ± 0.0005:

| submission     | central private LB | worst case (-1σ) |
|----------------|--------------------|-------------------|
| Primary        | ~0.97985           | ~0.97935          |
| Current hedge  | ~0.97939           | ~0.97889          |
| 3-way hedge    | ~0.97981           | ~0.97931          |

Primary still beats hedge in central case by ~0.00045. Switching to
3-way (F1) shrinks worst-case loss while keeping best-case cost
negligible.

### F5 (MEDIUM) — OOF→LB gap pattern: widening then sudden flip

```
recipe_full_te                  0.97967 → 0.97939  gap +0.00028
recipe × pseudo_s1              0.98012 → 0.97998  gap +0.00014
3-way (rec/s1/s7)               0.98029 → 0.98005  gap +0.00024
+ RealMLP α=0.20 + nonrule_iso  0.98061 → 0.98008  gap +0.00053
+ xgb_metastack__iso α=0.30     0.98084 → 0.98094  gap −0.00010   ← inversion
```

Gap widens monotonically, then suddenly inverts at the meta-stacker
step. "CV pessimism" requires meta-stacker to capture signal CV
specifically underestimates while nothing earlier in the stack does so.
Possible (meta sees cross-component disagreement features) but
unverified. Alternative: lucky public-LB draw on a heavily-tuned final
layer (α grid + iso-on-full-OOF both add small selection biases). This
pattern is the load-bearing argument for the F1 hedge swap.

### F6 (LOW) — Bias mismatch between recipe and meta-stacker prob scales

The fixed bias `[1.4324, 1.4689, 3.4008]` was tuned on
`oof_recipe_full_te`. Reusing it on a log-blend that includes
iso-calibrated meta-stacker output is correct under the binhigh rule
(don't retune bias when adding components — proven LB-regressing
multiple times in the log) but means the operating point isn't fully
optimal for the actual blend distribution. **No action.**

### F7 (NO ACTION) — Recipe FE pipeline is leak-free

OrderedTE correctly excludes current row (`cum_sum_excl = cum_sum_incl
- y_bin` at `recipe_ote.py:51`). FREQ uses train+test+orig value-counts
(target-agnostic). ORIG_mean/std uses ORIG target only (external dataset).
Per-fold `te.fit(X_tr_shuf, ...)` then `te.transform(X_va)` is correct.

Sub-agent #2's flag of `recipe_features.py:119` (drop digit cols
constant on test) as a "leak" is **wrong**. Dropping a feature with
zero variance on test cannot leak labels — model trained without it
produces identical test predictions. Metadata access ≠ target leakage.

GBY pre-fit and pseudo-label global bias (also flagged by sub-agent #2)
are real but: GBY is OFF in the LB-best primary (`GBY=""` default), and
pseudo-label bias is empirically documented (stage-2 LB-regressed,
seed-7 labeler 2-way LB-regressed; team uses τ=0.98 stage-1 only).

### F8 (NO ACTION) — Test-prediction averaging in `recipe_full_te.py`

Line 403: `test_pred += model.predict_proba(...) / N_FOLDS`. Each fold
contributes 1/5, each fold's model uses a different OTE encoding. Correct.

## Recommendations

**Do**:
- Swap hedge → `submission_3way_recipe025_s1035_s7040.csv` (LB 0.98005,
  premium -0.00089). Sidesteps meta-stacker layer.
- Optional alternative: `submission_recipe_full_te_catboost.csv` for
  model-family diversity at same LB cost as current hedge.

**Don't**:
- Don't burn LB probes on speculative variants — selection bias is
  meaningful at probe 30+. Reserve probes for pre-registered
  hypotheses.
- Don't retune log-bias on primary. Current bias works (LB > OOF).
  Retune now = OOF overfit risk per binhigh rule.
- Don't try to "fix" iso-on-full-OOF before deadline. Fix requires
  re-running greedy + new LB probe; not worth the variance of changing
  primary.

## Future-comp playbook additions

- **Per-fold isotonic** when iso-calibrated component is a stacking
  input. Current pattern is the standard Kaggle approach but inflates
  OOF by ~0.0001-0.0003 on a 3-class macro-recall problem.
- **Pre-register hedge selection criterion** (% argmax disagreement
  with primary, not just "second-best LB submission"). Avoids the
  intra-primary-diversity-as-hedge fallacy.
- **Track cumulative LB probes** and bake max-of-N selection-bias
  correction into expected-private-LB estimates after probe 10+.
